The Problem
Before — verbose SDK calls
// 3 lines per span
ctx, span := tracer.Start(ctx, "orders.process",
    trace.WithAttributes(attribute.Int("id", 42)))
defer span.End()

// verbose metrics
counter.Add(ctx, 1,
    metric.WithAttributes(
        attribute.String("method", "card"),
    ))

// manual test setup (15+ lines)
sr := tracetest.NewSpanRecorder()
tp := sdktrace.NewTracerProvider(
    sdktrace.WithSpanProcessor(sr))
t.Cleanup(func() {
    _ = tp.Shutdown(context.Background())
})
reader := sdkmetric.NewManualReader()
mp := sdkmetric.NewMeterProvider(
    sdkmetric.WithReader(reader))
// ...more boilerplate...
After — ergonomic wrappers
// 1 call, auto span lifecycle
otel.Trace(ctx, tracer, "orders.process",
    func(ctx context.Context, span trace.Span) error {
        span.SetAttributes(attribute.Int("id", 42))
        return processOrder(ctx, 42)
    })

// simplified attributes
counter.Add(ctx, 1,
    attribute.String("method", "card"))

// block timing with unit conversion
hist.Time(ctx, func(ctx context.Context) error {
    return chargeOrder(ctx)
})

// 1-line test setup
h := oteltest.Setup(t)

Attribute simplification. All methods accept ...attribute.KeyValue directly. The wrapper converts to metric.WithAttributes(attrs...) internally, eliminating one level of nesting from every recording call.
trace convenience
Trace() Function
Signature
func Trace(
    ctx    context.Context,
    tracer trace.Tracer,
    name   string,
    fn     func(ctx context.Context,
               span trace.Span) error,
    opts   ...trace.SpanStartOption,
) error
Flow
1. tracer.Start(ctx, name, opts...)
2. defer span.End()
3. err := fn(ctx, span)
4. If err != nil:
    span.RecordError(err)
    span.SetStatus(codes.Error, ...)
5. Return err

Zero breaking changes. All additions are new exports in the root package + a new sub-package. Existing Setup(), NewMiddleware(), NewLogger(), DetachedContext(), TraceHandler, and FanoutHandler are untouched.
extractHistogramUnit. Uses the public metric.NewFloat64HistogramConfig(opts...) API to read the unit from user-provided options. This avoids requiring users to pass the unit twice (once to the SDK, once to our wrapper). If the SDK changes this API, it will be a compile-time error — not a silent runtime failure.
Histogram.Time() sets span error. This is intentionally different from otel-rb (which does pure timing). Timing an operation and recording its failure are almost always done together in Go. Users who want pure timing can use Record() with their own time.Since().
