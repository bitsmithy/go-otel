tracing
Tracing API
Telemetry.trace(name, attrs:) { |span| }
Creates span, yields it, auto-finishes on block exit. Nested calls create child spans automatically. 1 line per span.
tracer.Start(ctx, name) Gap
Returns (ctx, span). Requires manual defer span.End(). 3 lines per span. No convenience wrapper exists yet.
metrics
Metrics API
Instrument Wrappers
Counter, Histogram, Gauge, UpDownCounter — each wraps OTel SDK instrument with ergonomic API.
  • counter.add(1, attrs)
  • histogram.record(42.0, attrs)
  • histogram.time(attrs) { block }
  • udc.increment / udc.decrement
  • Telemetry.time(name) { block }
Dispatch & Caching
Handle form returns cached instrument. Fire-and-forget records immediately. Instruments cached by [type, name].
Raw SDK Only Gap
Users create instruments manually via meter.Int64Counter(...), handle errors, store references, use verbose attribute API.
  • counter.Add(ctx, 1, metric.WithAttributes(...))
  • No histogram timing helper
  • No increment/decrement semantics
  • No instrument caching layer

logging
Logging
Dual Emission Parity
Logger emits to OTel + Rails.logger. LogBridge intercepts Rails.logger calls. TraceFormatter decorates with trace IDs.
Dual Sink Parity
FanoutHandler sends to OTel + stderr. TraceHandler injects trace_id/span_id. NewLogger() for additional instances.
testing
Test Support
Telemetry.test_mode!
Suppresses exporters. In-memory exporters for assertions. Telemetry.reset! clears state between tests. Auto-activates in test env.
No test helpers Gap
Tests manually configure in-memory exporters. No shared setup utility. Each test repeats boilerplate for span/metric inspection.
implementation plan
What to Build
P0 Metric Wrappers
Counter, Histogram, Gauge, UpDownCounter — Go structs wrapping SDK instruments. Simplified Add/Record accepting ...attribute.KeyValue directly. Histogram.Time() for block timing with unit conversion. UpDownCounter.Increment/Decrement.
P0 Trace Convenience
Trace(ctx, tracer, name, fn, opts...) — creates span, defers End, calls fn, sets error status on error. Reduces 3-line span lifecycle to 1 call.
P1 Test Helpers
oteltest.Setup(t) — returns in-memory tracer, meter, logger + span/metric accessors. Auto-cleanup via t.Cleanup. No exporter boilerplate.
N/A Rails Integration
LogBridge, TraceFormatter, auto-middleware insertion — Rails-specific features that don't apply to Go. go-otel achieves equivalent functionality through slog handler composition.
Design principle: Go wrappers should simplify, not hide. Constructors return (T, error). Methods accept context.Context. Attributes use attribute.KeyValue. No implicit caching or module-level state — users create instruments explicitly and manage their own references, the Go way.
