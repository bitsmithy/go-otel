File Purpose Type

Implementation Notes
jobs + groups over legacy priority
Lefthook's jobs feature (v1.10.0+) with nested group blocks cleanly separates the piped fixers stage from the parallel checks stage. The legacy commands + priority approach cannot run same-priority commands in parallel — they always execute sequentially.
gofumpt -d over -l in CI
gofumpt -d . outputs a unified diff of formatting issues and exits non-zero when files need formatting. -l only lists filenames without showing what changed — less actionable for developers reviewing CI failures.
govulncheck is CI-only
govulncheck is slow and requires network access to fetch vulnerability data from the Go vulnerability database. Not suitable for a pre-commit hook. Installed at runtime in CI via go install.
No .golangci.yml needed
Following the sibling alchemist project's pattern: default golangci-lint settings are used. A configuration file can be added later if specific linters need tuning or disabling.
